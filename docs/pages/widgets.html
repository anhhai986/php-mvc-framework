Development - Widgets

Generally widget is a class that inherits from CWidget helper class. It is a helper "component" that is mainly used for presentational purposes. A widget is usually embedded in a view script to generate a complex, yet self-contained user interface. For example, the CFormView widget can be used to render a complex form user interface, the CMenu widget can be used to render a different menus etc. Generaly widgets may be used in controllers, views or in template files.

To use a widget, you have to call it in a following way: 
<?php
echo CWidget::create('CMessage', array(
    'info',
    'Click <a href="login"><b>here</b></a> to log into the system as administrator.'
));
or 
<?php
$output = CWidget::create('CMenu', array(
    'type'=>'horizontal',
    'class'=>'user_menu',
    'items'=>array(
        array('label'=>($viewRightMenu) ? 'Back to Admin Panel' : 'Home', 'url'=>'authors/index'),
        array('label'=>'Logout', 'url'=>'login/logout'),
    ),
    'return'=>true
));
Here the full list of current system widgets:
  • menu draws menu
  • breadCrumbs draws breadcrumbs
  • message draws messages on the screen
  • form draws HTML forms
  • formValidation performs validation of HTML forms
  • pagination draws pagination control
You may always check the CWidget class in core/helpers/CWidget.php to learn the syntax of each widget or see the examples of code.
